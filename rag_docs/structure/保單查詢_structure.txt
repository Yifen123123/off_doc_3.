【文件用途】
本說明文件用來描述「保單查詢」類型公文的基本結構與應擷取欄位。此類公文通常由法院、消債案件承辦庭、金融機構，或經由壽險公會函轉，向各壽險公司查詢債務人、聲請人或相關人名下之保險契約資料。

本類型公文可能有兩種主要形式：
1. 直接查詢：法院或機關直接向壽險公司發函查詢。
2. 函轉查詢：某機關（例如壽險公會）將法院來函「函轉」給壽險公司，請保險公司依原函內容辦理並逕復原發文機關。:contentReference[oaicite:0]{index=0}

【典型整體結構】
保單查詢公文通常包含下列區塊（格式與順序可略有差異）：

1. Title與發文機關：
   - 例如「臺灣○○地方法院民事執行處 函」、「中華民國人壽保險商業同業公會 書函」等。
   - 有時會同時出現「函轉者」與「原發文機關」（例如公會函轉法院來文）。:contentReference[oaicite:1]{index=1}

2. 聯絡資訊：
   - 地址、承辦人、電話、傳真、電子郵件等。

3. 受文者：
   - 通常是「○○人壽保險股份有限公司」或「各會員公司」等。

4. 發文日期與發文字號：
   - 「發文日期：中華民國○○年○月○日」
   - 「發文字號：……」

5. 主旨：
   - 通常說明為「函請查明某人保險資料」、「函轉某法院函請查明保險資料，請依附件辦理」等。

6. 說明：
   - 條列「一、二、三、…」說明：
     - 法源依據（如：消費者債務清理條例第9條、第10條等）。:contentReference[oaicite:3]{index=3}
     - 受理案件案號（消債更、消債清等）。
     - 查詢對象（債務人、聲請人、或特定被保險人）。
     - 要求查詢的保單資料內容與範圍。
     - 如為函轉，會說明「依某法院函辦理」「請逕復該院」等。:contentReference[oaicite:4]{index=4}

【共通欄位說明】
以下欄位在多數保單查詢類公文中都應擷取（與其他公文類型共通）：

1. 發文機關：
   - 抬頭中出現的發文主體，例如「臺灣高雄地方法院民事執行處」、「中華民國人壽保險商業同業公會」。

2. 發文日期：
   - 明確標示為「發文日期：中華民國○○年○月○日」的行。

3. 發文字號：
   - 以「發文字號：」開頭，後接年度及案號，例如「雄院國113消債清保233字第1134077566號」、「南院揚民裡113消債更670字第1141000733號」。

4. 承辦資訊／電話：
   - 「承辦人：○○書記官」與「電話：(xx)xxxxxxx轉xxxx」等行。

5. 查詢對象（姓名與身分證號）：
   - 通常在說明中，以「聲請人○○○（身分證統一編號：XXXXXX）」、「債務人○○○」、「○○○（身分證統一編號：XXXXXX號）」等形式出現。
   - 若同時提及多位當事人，且僅能填一組資料時，優先擷取主要債務人或說明中被查詢之對象。

【本類型專屬欄位說明】

本文件重點在於協助模型判斷「這份公文在查什麼」，特別針對以下四項查詢需求：

1. 契約變更項目  
2. 要保人變更  
3. 保險金給付紀錄  
4. 保單借還款記錄  

同時也需處理「函轉」型式公文中之函轉關係與雙方聯絡資訊。

【一、一般保單查詢類的查詢項目判斷】

在說明段落中，可能出現下列敘述，需對應到各查詢項目欄位：

1. 契約變更項目（保單變更資料）  
   - 常見用語：
     - 「保單變更資料影本」  
     - 「所有保單變更資料」  
     - 「契約變更項目及日期」  
   - 例如：「請提供某保單於某期間內之所有保單變更資料影本乙份。」:contentReference[oaicite:8]{index=8}  
   - 若文字中提到「變更繳別、變更保額、增減附約、變更受益人等」，都可視為契約變更項目範疇。

2. 要保人變更  
   - 常見用語：
     - 「保單若曾變更要保人，請提供變更日期。」:contentReference[oaicite:9]{index=9}  
     - 「請說明要保人是否曾變更及相關紀錄。」  
   - 此類句子明確要求「要保人變更」相關資訊時，應在「要保人變更」欄位標記為有查詢，並整理出：
     - 是否有變更  
     - 變更日期  
     - 如有文字：可包含變更前後要保人姓名。

3. 保險金給付紀錄  
   - 常見用語：
     - 「如有領取解約金、保險給付、生存保險金或保險契約滿期領回保險金，請提供領取金額、時間、匯入帳戶等明細。」:contentReference[oaicite:10]{index=10}  
     - 「請提供保險金給付紀錄」、「失能或傷病請領保險給付紀錄」等。:contentReference[oaicite:11]{index=11}  
   - 當文字要求提供任何「解約金、保險給付、生存金、滿期金」等領取紀錄，應視為保險金給付紀錄查詢範圍。

4. 保單借還款記錄  
   - 常見用語：
     - 「如有保單借款，請並敘明保單借款數額、借款日期、扣除保單借款後尚餘之解約金數額。」:contentReference[oaicite:12]{index=12}  
     - 「保單借款及還款資料」  
   - 只要要求提供「保單借款金額、借款日期、借款餘額、扣除保單借款後解約金數額」等相關內容，即視為「保單借還款記錄」之查詢範圍。

在輸出格式中，可以用一個區段「查詢項目」列出上述四類中實際被要求的項目，例如：
- 查詢項目：契約變更項目、要保人變更、保險金給付紀錄、保單借還款記錄

若文中並未提及某項，則在對應項目中視為「未明確查詢」即可。

【二、查詢保單範圍與期間】

1. 查詢保單範圍：
   - 例如：
     - 「為要保人或被保險人之所有保險契約（含有效、停效、終止、解約保單）」:contentReference[oaicite:13]{index=13}  
     - 「是否向各保險公司投保商業保險？若有，請提供要保人、被保險人、保險公司、險種、保額、契約生效日期、有效期間、保費金額、終止後可領回之金額等。」:contentReference[oaicite:14]{index=14}  
     - 「就某一保單號碼於某期間內之所有保單變更資料。」:contentReference[oaicite:15]{index=15}  

2. 查詢期間：
   - 例如：
     - 「自民國111年9月起迄至本函送達之日止，如有領取解約金、保險給付、生存保險金或保險契約滿期領回保險金……」:contentReference[oaicite:16]{index=16}  
     - 「自某年某月某日起至某年某月某日止間之所有保單變更資料。」:contentReference[oaicite:17]{index=17}  
   - 若完全無期間字樣，則於「查詢期間」欄位填「無明確記載」。

【三、函轉型公文的特殊欄位】

部分保單查詢為「函轉」形式，例如：壽險公會收到法院來文後，將之轉發給各會員公司，請保險公司逕復法院。:contentReference[oaicite:18]{index=18}  
此時，需額外擷取以下資訊：

1. 函轉關係：
   - 轉出機關（函轉者）：
     - 例如「中華民國人壽保險商業同業公會 書函」為發文者，其主旨中常見「函轉某法院函請查明…」字樣。:contentReference[oaicite:19]{index=19}
   - 原發文機關：
     - 例如「臺灣臺南地方法院民事庭 函」，公會說明中會提及「依據某法院某號函辦理」，並附上原函全文。:contentReference[oaicite:20]{index=20}  

2. 雙方聯絡方式：
   - 對函轉者（如壽險公會）：
     - 地址、承辦人、電話、電子郵件等。
   - 對原發文機關（如法院）：
     - 地址、承辦人、電話、傳真等。
   - 在最終輸出時，可整理成：
     - 「函轉機關聯絡資訊：……」
     - 「原發文機關聯絡資訊：……」

3. 最終應回覆對象：
   - 多數函轉公文會明寫：
     - 「請依函文所揭事項辦理並逕復該院，毋須副知本會。」:contentReference[oaicite:21]{index=21}  
   - 此時，儘管受文者是保險公司，最終回覆對象仍為「原發文法院」，模型在整理「查詢項目／特別指示」時可加註此點。

4. 最後需要查詢的資料是什麼：
   - 實際查詢內容通常詳載於「原發文機關的說明」中（例如法院函的說明二、說明三）。:contentReference[oaicite:22]{index=22}  
   - 在函轉情境中，應優先根據原法院來函中列出的查詢事項來判斷：
     - 是否查詢保單存在與否
     - 是否查詢保額、險種、生效日、有效期間
     - 是否查詢可領回金額（解約金等）
     - 是否查詢契約變更、要保人變更、保單借款、保險金給付等紀錄

【判讀原則與模糊狀況處理】

1. 查詢項目四大類（契約變更項目、要保人變更、保險金給付紀錄、保單借還款記錄）只要有文字明確要求提供相關資料，即視為「有查詢」，應列入查詢項目中；未提及者不需臆測。
2. 函轉型公文中，若壽險公會主旨較短，實際查詢內容多半藏在附件的法院函中，模型應優先參考原法院函的說明段落。
3. 如公文中對期間、保單種類或對象界定不清，則在對應欄位填「無明確記載」，避免擴張解釋。
4. 若同時涉及多位查詢對象或多張保單號碼（例如同時對不同保險公司查詢），可在「查詢保單範圍」中加以說明，但主欄位仍以整體查詢要求為主。
5. 所有整理結果應貼近原文用語，如需重組句子，應避免改變原始意思或增添未載明之內容。


【輸出格式要求】
請依下列格式整理結果（若無明確資訊，請填「無明確記載」）：

發文機關：
發文日期：
發文字號：

一、來函資訊  
    速別：
    附件：
    承辦資訊（承辦人、電話）：

二、查詢對象  
    (1) 姓名：
    (2) 身分證號：

三、查詢項目（依公文內容會有多項）：
    - 契約變更資料：
    - 要保人變更紀錄：
    - 保險金給付紀錄：
    - 保單借還款紀錄：
    - 是否有函轉資訊（由誰轉來、原文號）：

四、法院或機關要求保險公司提供之資料：
    - 應回覆資料摘要：
    - 是否需要於期限內回覆：

五、其他備註：
